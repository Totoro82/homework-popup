# Comment Style

- English, informal, short (1-15 words)
- Only `//` inline comments, no block comments
- Explain the "why", not the "what"
- Explain variable purpose when name isn't obvious (e.g. `int low = totalCents / n; // base amount everyone should pay`)
- Use abbreviations (ppl, btw, etc.)
- Use arrows for mappings (e.g. `// value -> depth`)
- Comment above blocks for strategy, inline for quick clarifications
- Don't comment obvious stuff
